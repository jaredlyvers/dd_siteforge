# Pre-1.0 CMS Punch List — Design

## Context

`dd_staticsite` is a TUI-based CMS for small static marketing sites (5–20 pages, single editor, static HTML export). The component editor, renderer, validator, and storage layers are already in place; this design captures the remaining gaps needed before the tool is trustworthy for real client work.

Target workflow: author edits site in TUI, exports static HTML, hosts on any static host. No multi-user, no live server, no CMS-in-production.

## Goals

- Author can create, delete, rename, and reorder pages without editing JSON by hand.
- Author can export and preview from inside the TUI.
- Author can pick images from a local source directory instead of pasting URLs.
- Author sees validation errors before export fails silently.
- Author cannot lose more than ~2s of work to a crash.
- Theme schema complies with `THEME_STRUCTURE_STANDARD.md`.

## Non-Goals

- In-app undo/redo. Git + manual backup file is the safety net.
- Live-reload preview or embedded HTTP server.
- Copy/paste components or reusable block templates.
- Multi-user, auth, or any server-side feature.
- Remote image CDN integration beyond pasting an external URL into a field.

## Feature Specs

### 1. Page CRUD

New key bindings, **scoped to when the Pages panel (`[2] Nodes`) is the active sidebar section** — i.e. dispatched only when `selected_sidebar_section == SidebarSection::Pages`. The same keys retain their existing meanings when the Layout panel is active. This requires the top-level key-match in `handle_event` to branch on `selected_sidebar_section` before running the Layout-scoped arms.

| Key | Action |
| --- | --- |
| `A` | Add page — opens template picker modal |
| `X` | Delete current page — confirmation modal |
| `u` | Undo last deletion — restores from session trash |
| `J` / `K` | Move current page down / up in the list |
| `r` | Rename — opens page-head title field in an edit modal |

**Template picker modal.** Four options: `Blank` (just `[HEAD]`, no nodes), `Hero-only` (one empty `dd-hero`), `Hero+Section` (empty hero + one empty `dd-section` with a single column), `Duplicate current` (deep clone of the currently selected page, title suffixed `(Copy)`). Enter confirms, Esc cancels. Focus lands on the newly inserted page.

**Delete guardrails.** Blocked when only one page remains (status bar: "Cannot delete last page"). Otherwise a modal prompts `Delete "<title>"? y/n`. On `y`, the page is removed from `site.pages` and pushed onto a session-local trash `Vec<Page>`.

**Session trash + undo.** `App.deleted_pages: Vec<Page>` (ephemeral, not persisted). `u` pops the last entry back into `site.pages` at the end of the list, selects it. Capacity capped at 20 entries to bound memory.

**Rename + slug coupling.**
- New pages: slug auto-regenerates from the title each time the title is edited.
- Once the site containing this page has been successfully written to disk at least once (manual save or autosave), the slug locks; further title edits no longer touch slug.
- To edit a locked slug, the page-head modal exposes a separate `Slug` field.
- Tracked with a per-page `slug_locked: bool` persisted in the Page JSON.

**Reorder.** `J` / `K` swap adjacent entries in `site.pages`. That order is the sitemap/render order emitted by `render_site_to_dir`.

### 2. In-TUI Export

**State.** Add `Site.export_dir: Option<String>` persisted in JSON. Path is interpreted relative to the site JSON file.

**Trigger.** `E` key from any panel.

**Flow.**
1. Run `validate_site(&self.site)`.
2. If errors: open the validation modal (see §4), blocking. User must dismiss; export does not run.
3. Resolve `site.export_dir`:
   - If set: use it.
   - If unset: open a single-field prompt pre-filled with `./web/`. On confirm, write to `site.export_dir` and mark the site dirty for autosave.
4. Resolve path to absolute relative to the site JSON file.
5. Call `render_site_to_dir(&self.site, &resolved_out)`.
6. Copy `./source/imgs/` recursively to `<out>/assets/imgs/` (see §3).
7. Status bar: `Exported N pages to <path>`.

Errors during rendering or copying surface in the status bar; no partial cleanup is attempted (next export overwrites).

### 3. Image Assets

**Convention.**
- Images live in `./source/imgs/` relative to the site JSON file.
- Stored field value format: `assets/imgs/<rel>` (output-relative) or any `https?://...` URL.
- Renderer emits the stored string verbatim into HTML (so `assets/imgs/foo.jpg` resolves against the deployed site root).
- On export, `./source/imgs/**` is copied recursively to `<export_dir>/assets/imgs/`.

**Picker.** All `FieldKind::Url` form fields in `editform.rs` gain a `Ctrl+P` action that opens an image picker modal.

**Picker UX.**
- Modal opens rooted at `./source/imgs/`.
- Top line: fuzzy filter input (any typed characters narrow the visible list by substring match across relative paths).
- Below: vertical list of entries in the current directory — subdirectories first, files second, each prefixed with `/` or the file icon. Hidden files omitted.
- Navigation: `j`/`k` move selection, `h` ascend to parent dir (clamped at `./source/imgs/`), `l` or Enter descend into subdir or select a file.
- On file selection: write `assets/imgs/<path-from-source-root>` into the active URL field, close modal.
- Esc cancels without changing the field.

**Validation hook (§4).** When a URL field value starts with `assets/imgs/`, the validator resolves `./source/imgs/<rest>` and reports a `missing-image` error if the file does not exist.

### 4. Validation

**Trigger.** `F3` key from any panel, or implicit call from the export flow (§2). Lowercase `v` and uppercase `V` are already bound in the Layout panel; `F3` avoids the conflict and reads well for "validate".

**Modal.**
- Title: `Validation — N errors`.
- Body: numbered list of errors, one per line. Long messages wrap.
- `j` / `k` / PgUp / PgDn scroll.
- `Enter` or `Esc` dismisses.

**Behavior.** Uses the existing `validate_site()` from `src/validate.rs`. Extended with the `missing-image` check described in §3.

### 5. Autosave + Backup

**Autosave.**
- Any model mutation (page edits, reorders, adds/deletes, FormEdit commits) marks the site dirty.
- A 2-second debounce timer starts/restarts on each mutation.
- When the timer fires and the site is still dirty, write to the current site JSON path.
- No autosave if no path is set yet (fresh `Site::starter()` session until user runs save-as).

**Manual `s`.**
- Writes the current site JSON to the active path.
- Additionally writes a byte-identical copy to `<path>.backup`.
- Clears the dirty flag and cancels any pending debounce.

**Load-time check.** On startup, if both `<path>` and `<path>.backup` exist and differ, the status bar shows: `Loaded state differs from last manual save (<backup mtime>)`. No automatic recovery prompt; user inspects and decides (git is the real recovery mechanism).

**Undo.** Not implemented. Documented in the TUI help that `s` + git + `<path>.backup` are the rollback tools.

### 6. Preview

**Trigger.** `p` key, on a row that maps to a specific page (any page-tree row).

**Flow.** Runs the full export flow (§2) including validation gate, then invokes the system browser on `<export_dir>/<current-page-slug>.html`.

**Browser invocation.**
- Linux: `xdg-open`
- macOS: `open`
- Windows: `start`

Failure to spawn the browser surfaces in the status bar but export is still considered successful.

### 7. Theme Compliance

Schema brought into alignment with `THEME_STRUCTURE_STANDARD.md`. Already shipped as part of this session:
- `folders`, `files`, `links` added to `AppTheme` struct.
- Parsed from `PaletteFile` with defaults matching `dd_staticsite_theme.yml` (`#64B4F5`, `#FFAF46`, `#FFA087`).
- Rendering wiring deferred: tokens remain unused in draw code until a natural place to color containers vs leaves vs URL text appears (e.g., image picker in §3 is a plausible first consumer — folder/file distinction in its list view).

## Architecture Notes

- `App` struct gains: `deleted_pages: Vec<Page>`, `dirty: bool`, `autosave_deadline: Option<Instant>`.
- `Site` model gains: `export_dir: Option<String>`.
- `Page` model gains: `slug_locked: bool` (defaulted via serde to `false` for existing sites).
- `src/storage.rs`: no schema migration needed — new fields are optional with defaults.
- `src/validate.rs`: add `missing-image` error variant and source-file resolution helper.
- `src/tui.rs`: new modal types (`TemplatePicker`, `ConfirmDelete`, `ValidationErrors`, `ImagePicker`); new event handlers; event loop ticks the autosave deadline.
- No changes to `src/renderer.rs` beyond possibly reading `site.export_dir` default.

## Testing

- Model/storage round-trips with new optional fields (existing sites load, re-save without losing data).
- Validation includes `missing-image` path test.
- TUI integration tests for: add page via each template, delete last page refused, delete + undo, reorder, rename locks slug, export success + export-blocked-on-validation, autosave debounce (simulated clock), backup file written on manual save.

## Open Questions

None at time of writing. Any encountered during implementation will surface as follow-ups on this spec.

## Rollout Order

1. Feature #1 (Page CRUD) — largest UX win, unblocks real authoring.
2. Feature #4 (Validation modal) — prerequisite for #2.
3. Feature #2 (In-TUI Export) — unlocks end-to-end TUI workflow.
4. Feature #5 (Autosave + backup) — safety net before the tool sees real use.
5. Feature #3 (Image assets) — quality-of-life, depends on export being robust.
6. Feature #6 (Preview) — final polish, depends on export.
7. Feature #7 — schema already shipped; wire visuals opportunistically during #3.
